Iterative Refinement of Numeric Parameters

    Greedy Hill Climbing


    • Reduce either number of errors or cost of errors
    • Add or Subtract percentage (2.5%) of parameter value
    • Keep change if it makes an improvement
    • Keep change with 0.5 probability if no change in cost
    • If no change on all parameters, increase percentage by 2.5%
    • More constrained than FOCL
    • Changes Numeric Parameter with fewer intermediate step
