Property Namespace

  • Flat Namespace

    • Table of names and values
    • Easy for server to maintain
    • Majority of current implementations are flat

  • Names are URIs

    • Prevents Collisions
    • Defined Extension Mechanisms
    • For Example: AIIM:Dublin:Author, http://www.ietf.org/standards/dav/source, etc.
