Downloading the Jimage Client

  Chimera client applications have been modified to provide the user with ability to utilize hypermedia functions. These include the ability to make anchors on portions of documents, connect such anchors to links, and to traverse links. The Jimage client application provides such capabilities in the form of an application that displays gif and jpeg images.

Jimage will function as an application that displays images even if you don't have Chimera on your system. In that case, you will not have the ability to use the hypermedia functions Chimera provides.

Things to have in place for Jimage

Jimage is written in Java and it requires that you have the following in place on your system.

  • A Java Interpretor compatible with JDK 1.1.5
  • Swing 0.7 (This is used to display Jimage and should be in your CLASSPATH variable.)

(See Preparing your System for Chimera for an example of setting swing for your system.)

Downloading Jimage

You may download Jimage from the following site:
Chimera 2.0 Download page
Jimage should be listed on the page.

Place Jimage anywhere you wish on your system. A good spot is a directory which you use to hold your Java programs.

Preparing to run Jimage on your system

Once you have downloaded Jimage, you need to add it to the your CLASSPATH. CLASSPATH is the environment variable used by the Java interpretor to look for Java byte-code.

To see how to add Jimage to the CLASSPATH follow the appropriate link below:

  • Setting CLASSPATH in Unix to include jimage  
  • Setting CLASSPATH in Windows NT to include jimage  

You now have the Jimage application downloaded and properly set to be run.
