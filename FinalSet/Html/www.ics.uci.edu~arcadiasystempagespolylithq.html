PolylithQ

  1. Name:
    PolylithQ

  2. Source:
    University of Colorado Arcadia Project.

  3. Brief description:
    PolylithQ is a re-implementation of the University of Maryland Polylith 2.1 system but using Q for its underlying IPC. It provides essentially the same functionality as Polylith 2.1 but adds the following capabilities.
    • Compatibility with other Q based distributed programs.
    • All connections direct but with no loss of functionality.
    • Richer type system.
    • Better Ada support.

  4. Evaluation against applicable general dimensions:

    1. Availability: commercial/licensed/public domain
      Public domain.

    2. Cost:
      None; it is available via anonymous FTP.

    3. Degree of support/maturity/testing/usage:
      This was constructed as an experiment to compare Q and Polylith. No support will be provided except that bugs will be fixed.

    4. Speed:
      Speed is comparable with Polylith 2.1.

    5. Computing platforms:
      It is known to work on Sparcs running Solaris 1. Required Support Systems: Q.

    6. Language compatibilities:
      It is written in C; an Ada interface is also available.

    7. Footprint:
      1. Source and binary: 2 MB (not including Q)

    8. Openness/integrability/source availability:
      Source is freely available.

    9. Extensibility:
      Unknown.

    10. Pedigree:
      Developed by the ARPA sponsored Prototech project.
