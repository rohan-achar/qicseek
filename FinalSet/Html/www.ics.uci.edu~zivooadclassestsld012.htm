The Class Diagram

  • Class diagram is the core of a Unified Model
    • Shows the logical static structure of a system
    • Shows system elements and relationships among them
    • Shows generic descriptions of possible systems
      • An object diagram shows particular instantiations of systems
      • It is possible to mix classes and objects
    • A class is drawn as a rectangular box with 3 compartments
      • Class name
      • List of attributes (optional types and initial values)
      • List of operations (optional argument lists and return types)
